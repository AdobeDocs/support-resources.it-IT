# Pagina di destinazione di Workfront Admin Console

Prepara l&#39;onboarding della tua organizzazione nell&#39;elenco di controllo per la [migrazione di Adobe Admin Console](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/admin-in-admin-console/prep-for-admin-console.html?lang=it):

**Elenco di controllo migrazione**

* Identifica il Adobe Admin Console desiderato dove desideri aggiungere Workfront.

  Se disponi già di altri prodotti Adobe, conferma la tua disponibilità ad aggiungere Adobe Workfront al tuo Adobe Admin Console corrente. Per ulteriori informazioni, utilizza [questo collegamento](https://helpx.adobe.com/it/enterprise/using/admin-console.html) per comprendere le best practice per la gestione dei prodotti Adobe.

* Per assistenza sulla distribuzione, fare riferimento al [collegamento seguente](https://helpx.adobe.com/it/enterprise/using/deployment-planning.html).
* Se utilizzi l’accesso Single Sign-On, dovrai collegare il nuovo Adobe Admin Console al provider SSO esistente.
* Assicurati che le e-mail duplicate vengano rimosse e che vengano utilizzati solo indirizzi e-mail validi.
* Se utilizzi integrazioni API personalizzate, verifica che gli utenti abbiano un indirizzo e-mail valido a cui possono accedere.

  Workfront consiglia di aggiornare tutte le integrazioni personalizzate in [utilizzare OAuth2](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/configure-integrations/create-oauth-application.html?lang=it).

Dopo aver configurato la console, fai riferimento a questo collegamento per vedere [come gestire gli utenti](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/add-users/create-manage-users/admin-console.html?lang=it) nel tuo Adobe Admin Console.
